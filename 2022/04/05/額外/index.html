Process（進程、程序、處理序、行程）

  • 一個process或應用程式可以有多個執行緒
  • 當primary thread執行結束後process就會結束
  • 對作業系統來說，它是資源分配的最小單位
  • 把一個cpu分成多個單位,分給不同的process來做
  • 可分為同步或非同步
  • 一個process會有一個unique的id叫做 process id
  • 有兩個屬性：heap 和 stack
    heap 的內容可和其他process的thread的內容共享,stack則不行

一個編譯好的程式會由os來分配事情到個地方（process的 heap,stack, processID,program center)

途中藍色小格子是thread

Thread （線程、執行緒）

在談thread之前要先了解process

  • 作業系統能夠進行運算排程的最小單位
  • 有自己的call stack,stack 可同步或不同步執行
    JavaScript 和Ajax

一個執行緒有四種狀態，且可以由開發人員決定不同的優先順序:

  • 執行中（Running）
    • 執行緒正在執行當中。
  • 暫停（Suspended）
    • 暫停已在執行當中的執行緒，但可以讓它resume
  • 被阻擋（Blocked）
    • 執行緒因某個特定的資源被其他執行緒占用時，會進入等待的狀態。
  • 終止（Terminated）
    • 執行緒被終止或停止，但無法再繼續執行。

Service:(kotlin)

:::info
為甚麼要使用service:
在只剩下一個thread時為了避免process被系統回收導致thread提前結束

BroadcastReceiver

  • 用來接收或處理broadcast intents的類別（class）
  • intentfilter攔截Intent所描述的動作且和攔截的Intent做匹配，IntentFilter攔截的項目有actions, categories,data。
  • 和一些手機本身的事件有關，例如連網路、充電、重新開機，利用BroadCastReceiver來做出相對應的反應
  • 可以獨立運作也可以綁activity。例如綁broadCastReceiver在activity上，在執行時檢查是否有連到網路

使用：

  1. 建立一個名叫Receiver且類別繼承BroadCastReceiver的class
  2. 在要註冊的activity裏宣告一個Receiver class的物件
  3. 註冊廣播：registerReceiver(reciever,filter)
  4. 取消註冊：unregisterReceiver(reciever)

Android App 生命週期


Activity 啟動到結束這期間會做callback方法。Callback包括：

  • onCreate():開啟應用程式時第一個執行，並只會執行一次
  • **onStart()**：onCreate做完後就會做onStart。會把UI 物件初始化
  • onPause():一個功能執行到一半暫時暫停，跑去做另一件事情,這個功能就會做onPause
  • **onResume()**：onPause回來繼續執行時做onResume
  • onStop():一個應用程式做完不用再做,就會做onStop把手機資源釋放[Ｍemory Management概念]
  • **onDestroy()**：整個應用程式結束時做，和onCreate一樣只會做一次

一件事情做完都會去做onPause和onStop
取消註冊沒有在用的事件也可以替電腦省電！
(https://developer.android.com/topic/performance/memory-overview)

oncreate和onstart差別

:::warning
:question: 一個app可以不用onstop?
如果沒有中途跑去做另外一件事情的話，他就不會呼叫

中途呼叫另一個activity：
onPause(1) -> onCreate(2) -> onStart(2) - onResume(2) -> onStop(1)
先暫停第一個，等第二個完成啟動時才stop
:::

    Android App 開發流程

  1. 需求：包含市場調查、顧客反應、焦點訪談及使用者測試，針對結果訂定相對應主題。
  2. 設計：所有的app相關設計（包含行銷部分）
  3. 開發：工程師開始寫app
  4. 測試:測試使用起來有沒有問題

  6. 行銷
