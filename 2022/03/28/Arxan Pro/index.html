保護流程

  1. 從Digital.ai取得防護腳本
  2. 安裝防護工具
  3. 對app做防護
  4. 對app做簽名
  5. 將防護過的app部署到end user

    前置作業

    必備環境

  • Python
    python版本需達到3
  • 開發工具
    根據app選擇開發工具。
    開發Android app: Android Studio 和 Android SDK
    開發iOS app: Xcode
    • 環境變數
      下載完Android Studio後，需要將Android SDK的路徑加入環境變數
      • macOS及Linux
        1. 在.bash_profile 檔裡加入
          export ANDROID_SDK_ROOT=/path/to/Library/Android/sdk/
          修改/path/to/Library/Android/sdk/處，改你的Android SDK安裝路徑
        2. 存檔
      • Windows
        1. 前往 控制台 > 系統與安全性 > 系統 > 右側進階系統設定>上方橫排選擇 系統 > 環境變數
        2. 在系統變數區塊點選編輯。如果沒有ANDROID_HOME這個變數，點選右側新增來建立系統變數
          名稱:ANDROID_HOME (只能是這個名字)
          路徑:Android SDK安裝路徑
          EX: C:\Users\USER_NAME\AppData\Local\Android\sdk
        3. 儲存

取得API Credential

  1. 前往 digital.ai developer 。右上角Manage>>選擇Web API Management

  2. 點選GENERATE NEW。在

  3. Description處輸入credential名稱。Entitlements 選擇 Product download.

  4. 複製底下的Consumer key和Consumer Secret key。在安裝防護工具時會使用到。

    取得license token

  5. 前往 digital.ai developer ，Protect>On Promise>license token

  6. 點選右上方 GENERATE NEW TOKEN 產生token(保護及安裝工具時會用到)

    下載防護腳本 protect-pro.py

  7. Product: 選擇App Production for mobile:Pro

  8. Development Platform：選擇All Platform

  9. 下載protect-pro.py腳本

protect-pro-samples.zip為範例apk
